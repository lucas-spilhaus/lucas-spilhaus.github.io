An analogy to describe JavaScript and its relationship to HTM and CSS

Think of HTML as a building of some sort, say a house or store. HTML is the layout and basic structure of the building. Now CSS comes and handles decoration and the design. CSS decides the color scheme of the building, the wallpaper, flooring, etc. Now JavaScript is the fancy stuff. JavaScript lets us control various aspects of the building, like closing curtains or turning lights off via your phone. JavaScript allows us to interact with the building how we please.

Explain the difference between accessing data from arrays and objects

An array contains various items under one variable name while an object contains items under keys. Accessing data from an array is as simple as the array name with the slot you want to access, however it is important to remember that arrays start at 0 rather than 1. So, to access the first slot in an array, we would use array[0].

As for accessing data from an object, we use dot notation. While slightly more complicated than arrays, dot notation is still very simple. To access data we would go nameA.nameB.nameC, similar to how we would with folders on a computer, but with periods rather than backwards slashes.

Explain what functions are and why they are helpful

Functions are a way to save code and call upon it later. For example, I could write a function that generated a number between 1 and 10 and then call it as many times as needed throughout my code. Functions allow us to not have to re-write the same piece of code each time we want to use it. On top of this, functions let us break our code into modules, making it easier to find and correct bugs and improving our readability.
